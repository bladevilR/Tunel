# 功能与补齐项梳理

日期：2026-05-07

## 页面重构口径

本轮按四个真实工作页面重构，不再把录入、评估、知识库、报告全部挤在同一屏：

- 总览工作台：展示资料源、知识块、站点索引、项目库、资料就绪度、补齐项总表和导出历史。
- 项目评估：承载标准输入表单、真实后端评估结果、站点上下文、推荐方案和评分矩阵。
- 知识库：承载资料源统计、关键词检索、分类筛选、资料源清单和知识库补齐项。
- 报告与补齐项：承载六章报告预览、真实导出文件链接、本项目补齐项和规则追溯。

## 试点交付状态区

总览工作台新增“试点交付状态”区域，用于把评审最关心的交付边界前置展示：

- 已完成：概括项目管理、评估、报告、知识库检索和真实数据读取等已接后端能力。
- 需外部资料：复用当前评估缺项、项目字段缺项和知识库未解析来源，不新增写死缺口。
- 不能自动完成：明确控规/接口/管线/产权等外部提资，以及 CAD/DWG/BIM、施工图审查、工程级图纸生成等一期不可自动完成事项。
- 区域内不提供新增操作按钮，避免把资料补齐、OCR、工程校核误表达为系统已可自动执行。

## 已做实功能

- 项目评估：`POST /api/evaluate`，按 PPT/评价因子表/20260507 反馈口径计算评分、等级、推荐方式。
- 项目保存：`POST /api/projects`，写入 `data/projects.json`。
- 项目读取：`GET /api/projects/{id}`，读取并重新评估保存项目。
- 项目删除：`DELETE /api/projects/{id}`，删除本地保存项目。
- 报告导出：`POST /api/export`，生成 Markdown、Word、JSON、评分明细 CSV、待补齐 CSV。
- 评价报告增强：后端生成不少于 10 章的综合研判报告，并在 JSON 中输出 `llmReviewContext`，供后续 LLM 接入做解释、复核和专家式建议。
- 导出历史：`GET /api/exports`，从 `exports/` 目录读取真实导出文件。
- 知识库摘要：`GET /api/knowledge`，返回知识库统计、规则卡、站点索引、资料源和未解析来源。
- 资料源清单：`GET /api/sources`，返回 `source_manifest.json` 与 `unparsed_sources.json`。
- 知识库检索：`GET /api/knowledge/search`，按本地知识块检索，不调用假数据。
- 站点上下文：评估结果中返回客流、出入口接口、202604 出入口开放和周边配套。

## 已清理或降级的假动作

- 移除原“方案示意预览”假图层，避免用户误以为系统已生成工程图。
- 移除静态状态堆叠，改为从 `knowledge_catalog.json`、`source_manifest.json`、`projects.json`、`exports/` 读取。
- 导出按钮改为后端生成真实文件后展示真实链接，并触发一个真实 Markdown 下载。
- 知识库分类筛选只过滤真实检索结果，不展示写死卡片。
- 项目库表格与项目选择器共用真实保存项目数据。
- LLM 综合判断框架只接收后端评分事实、引用依据、维度拆解、方案比选和风险复核点，不允许模型自行改分、改等级或把缺失资料补成确定事实。

## 待补齐项

### 知识库资料补齐

- `轨道站点出入口周边空间一体化设计指引(1).doc`：已转换为同名 `.docx` 并重建知识库，旧版 Word 不再计入未解析缺口。

- 早期豆包报告：已作为低优先参考入库，不参与规则裁决；后续无需按规则源补齐。
- `~$车站TOD级别划分整理.xlsx`：Office 临时锁文件，无业务价值，继续忽略。

### 项目资料补齐

系统会按当前项目自动标注以下缺项：

- 站点级预评估项目：240 个站点已接入 TOD、客流、出入口、接口和周边配套线索；但不是 240 个具体地块提资，因此用地功能、开发强度、地下空间仍按正式评分输入标注为待补齐。
- 附件资料：控规图则、规划条件、概念方案、附件目录。
- 邻近出入口：拟接驳出入口编号、空间位置和周边界面。
- 接口条件：接口位置、标高、非付费区条件、运营管理界面。
- 规划指标：容积率、建筑密度、限高、用地边界。
- 地下空间说明：地下层数、公共开放功能、可接驳空间。
- 特殊约束：地下管线、风貌管控、建设时序、产权和施工界面。
